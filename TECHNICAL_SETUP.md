# Technical Setup

This repository contains a neutral SvelteKit starter scaffolded with the current Svelte CLI, TypeScript, and Tailwind CSS using the official Tailwind Vite plugin integration. It is packaged as a static-site template for GitHub Pages, with Docker supporting development workflows rather than production hosting.

## Stack

- Node 22
- SvelteKit
- Svelte 5
- `@sveltejs/adapter-static`
- Tailwind CSS 4
- npm
- VS Code Dev Containers
- Docker for local development support
- GitHub Pages deployment via GitHub Actions

## Open In The Dev Container

1. Open the folder in VS Code.
2. Run `Dev Containers: Reopen in Container`.
3. Wait for the container lifecycle hooks to finish.

The Dev Container will:

- build the `dev` target from the root `Dockerfile`
- install dependencies with `npm install`
- sync your host `~/.gitconfig` into the container
- preserve any existing container `user.name` and `user.email`
- start one detached SvelteKit dev server
- forward the running app port and open it in the browser when VS Code forwards it

The included VS Code extension recommendations stay focused on Svelte, Tailwind CSS, and Dev Containers so the template does not assume any specific AI tooling.

## Development

If you are working outside the container, this project expects Node `22`. The included [`.nvmrc`](./.nvmrc) file reflects that version.

The app follows standard SvelteKit commands:

```sh
npm run dev
npm run build
npm run preview
npm run check
```

The dev container startup script prefers port `5173`. If that port is already in use in the container, it automatically selects the next available port and prints the actual port.

## Static Build And Deployment

Production output is a static site generated by `@sveltejs/adapter-static`.

- `npm run build` writes the static output to `build/`
- `actions/configure-pages` prepares the workflow for GitHub Pages deployment
- `.github/workflows/deploy-pages.yml` uploads `build/` as the GitHub Pages artifact
- `BASE_PATH` is set during the workflow so the built site works for both `username.github.io` repositories and project pages such as `/repo-name`
- the deploy jobs run automatically only on the repository's default branch, while `workflow_dispatch` remains available for manual deploys

## Docker

Build and run the development container image directly:

```sh
npm run docker:dev:build
npm run docker:dev:run
```

The standalone Docker dev runner prefers host port `5173`. If that port is already in use on your computer, it automatically selects the next available port up to `5190` and starts Vite on that same port inside the container.

## Notes

- The starter app content is intentionally generic so the repository can be reused as a neutral template.
- Vite is configured to listen on `0.0.0.0` so the app is reachable from the host through VS Code port forwarding.
- The root `Dockerfile` is used for the Dev Container and optional local Docker-based development.
- GitHub Pages is the intended production deployment target for this template.
